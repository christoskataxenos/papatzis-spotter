# Guidelines — AiSlop-Generation-finder (AI Agent Rules)

## 🎯 1. Το Όραμα
Κάθε γραμμή κώδικα σε αυτό το project πρέπει να είναι το αντίθετο του "slop". Πρέπει να φαίνεται ότι έχει γραφτεί από άνθρωπο που νοιάζεται για την λεπτομέρεια, την αναγνωσιμότητα και την ποιότητα.

---

## 🐍 2. Python Coding Style
- **Snake Case**: Πάντα για μεταβλητές και συναρτήσεις.
- **Μικρά Modules**: Όχι τεράστια αρχεία. Κάθε αρχείο έχει έναν ρόλο.
- **Σχόλια**: Πάντα στα **Ελληνικά**, με φυσικό λόγο. 
    - *Κακό*: `# Update generic score`
    - *Καλό*: `# Ενημερώνουμε το σκορ με βάση τα βάρη που ορίστηκαν στην αρχή`
- **Typing**: Χρήση Type Hints παντού.

---

## ⚛️ 3. Frontend & React Style
- **Naming**: Όχι `data`, `item`, `stuff`. Χρησιμοποιούμε συγκεκριμένα ονόματα (`analysis`, `finding`, `pillar`).
- **Components**: Functional components μόνο, χωρισμένα σε μικρά κομμάτια.
- **CSS**: TailwindCSS v3 με custom design tokens. Όχι inline styles αν δεν είναι απαραίτητο για το logic.

---

## 🦀 4. Rust & Tauri Style
- **Safety First**: Όχι `unwrap()` χωρίς λόγο. Πάντα handle τα errors και επιστροφή Result.
- **Clean Bridges**: Οι εντολές (commands) πρέπει να είναι απλές και να μεταφέρουν data, όχι περίπλοκο logic.

---

## 🤖 5. Κανόνες για το AI Agent
Σαν AI Assistant, όταν δουλεύεις σε αυτό το repo, πρέπει:
1. **Να αποφεύγεις το "Perfect Symmetry"**: Ο κώδικας που γράφεις πρέπει να έχει οργανική δομή. Μην χρησιμοποιείς τα ίδια templates παντού.
2. **Να αυξάνεις την Εντροπία**: Χρησιμοποίησε μεταβλητότητα στο μήκος των γραμμών και στην ονοματολογία. Απόφυγε την υπερβολική "προβλεψιμότητα".
3. **Να γράφεις ποιοτικά σχόλια**: Τα σχόλια πρέπει να εξηγούν το **γιατί** (intent), όχι το **τι**. Μην χρησιμοποιείς GPT buzzwords.
4. **Premium Αισθητική**: Κάθε νέο UI component πρέπει να είναι υψηλής ποιότητας και να ταιριάζει με το `design.md`.
5. **Humanity Detection**: Αν εντοπίσεις ότι ο δικός σου κώδικας μοιάζει με slop, αναδιαμόρφωσέ τον για να έχει περισσότερο "χαρακτήρα".
